# 👶 婴儿哭声标注系统 (BabyCryLabeler) 极速使用指南

欢迎使用**婴儿哭声精准标注系统**！为了让您在最快时间内上手并高效率完成哭声数据集的标注，我们为您准备了这篇“通俗易懂、3分钟上手”的极速使用指南。

---

## 🛠️ 本地下载与环境配置

在浏览器中开始使用前，请确保您已经在电脑上下载了系统源码，并按照以下极简步骤完成 Node.js 运行环境的配置与启动：

### 1. 安装 Node.js 运行环境
* 本系统基于前端 React 与后端 Express 构建，需要您的电脑安装 **Node.js**（推荐使用 v18 或更高版本）。
* **下载地址**：前往 [Node.js 官网 (nodejs.org)](https://nodejs.org/) 下载适合您系统的 LTS（长期支持）版本安装包（Windows / macOS / Linux），并一路点击“下一步”默认安装即可。
* **验证安装**：打开命令行/终端（Terminal），输入以下命令。若正确输出版本号（如 `v18.x.x`）则表示安装成功：
  ```bash
  node -v
  npm -v
  ```

### 2. 安装依赖并一键启动
1. **解压或克隆**系统源码到您的本地电脑任意文件夹中。
2. 打开您的命令行/终端，并切换到该项目的根目录。
3. 运行以下命令，系统会自动安全下载安装项目所需的全部依赖包（如 React, Tailwind 等）：
  ```bash
  npm install
  ```
4. 依赖安装完成后，运行以下命令一键启动本地服务器：
  ```bash
  npm run dev
  ```
5. 当看到控制台打印出 `Server running on http://localhost:3000` 后，本地环境就已经完美运行了！

---

## 🌐 如何在浏览器中访问与设置

系统部署成功后，您可以通过浏览器极速开启工作。为了获得最极致、不卡顿的标注体验，请注意以下几点：

### 1. 访问系统网页
* **访问地址**：在浏览器地址栏输入 **`http://localhost:3000`** 即可。
* **适用场景**：这是您本地运行系统的标准地址（通过在系统目录下运行 `npm run dev` 启动）。每个人在自己电脑上运行后，均可以通过该地址独立访问系统，数据完全保存在您本地。
* **本地优势**：本地运行**完全离线、响应极快**，且支持输入本地绝对路径（如 `D:\dataset\cries`）直接扫描并加载电脑硬盘上的数万条音频！


### 2. 授权浏览器“自动播放声音”权限 (关键 🔑)
* **常见现象**：现代浏览器为了防止网页广告骚扰，**默认禁止网页在未经用户点击的前提下自动播放声音**。这可能导致您刚打开网页时听不到声音。
* **解决方法**：
  * **随便点一下**：只要在网页任意地方用鼠标点击一下，激活浏览器与页面的交互，系统便能立刻恢复自动连播声音。
  * **永久允许（推荐）**：点击浏览器地址栏左侧的“🔒锁头”或“🌐图标”，选择 **“网站设置”**，将 **“声音 (Sound)”** 权限设为 **“允许 (Allow)”**。

## 🚀 第一步：导入你的哭声音频

开始标注前，需要先让系统“看到”并“加载”您的哭声音频文件。系统提供了三种极其简单、高效的导入方式，您可按需选择：

| 导入方式 | 操作步骤 | 适用场景 |
| :--- | :--- | :--- |
| **方式 A：一键拖拽（极力推荐）** | 直接从您的电脑文件夹里，用鼠标**框选数百条音频**（支持 `.wav`, `.mp3`, `.m4a` 等所有主流格式），**一键拖入**网页中心灰色的“拖拽音频文件到此处”区域。 | 适合日常人工打标、大批量文件快速导入 |
| **方式 B：自动生成模拟演示** | 如果您手上暂时没有现成的音频，只需点击网页上的 **“自动合成本地模拟演示音频 (Demo)”** 按钮。系统会在 1 秒内为您在后台合成 5 首模拟婴儿哭声音频。 | 适合首次使用、快速熟悉系统和快捷键 |
| **方式 C：扫描本地文件夹绝对路径** | 点击右侧的“设置”按钮打开面板，在“扫描指定路径”框中输入您电脑里存储音频文件夹的完整绝对路径（例如 `D:\dataset\cries`），然后点击**“扫描”**。 | 适合本地运行、文件夹内包含数千首音频的大型项目 |

## 🎧 第二步：极速人工标注（听音打标）

导入音频后，播放器会自动开始播放第一首哭声。您无需频繁移动鼠标去点按钮，系统已为您全方位打通了**“键盘流无缝打标”**：

### 1. 键盘快捷键（推荐，熟练后每首仅需 1 秒）
* 按 **`1`** ➔ 标注为 **“饥饿”** 👶
* 按 **`2`** ➔ 标注为 **“不舒服”** 🧷
* 按 **`3`** ➔ 标注为 **“犯困”** 💤
* 按 **`4`** ➔ 标注为 **“需要拍嗝”** 🍼
* 按 **`5`** ➔ 标注为 **“烦躁/情绪”** 😭
* 按 **`空格键 (Space)`** ➔ **暂停 / 继续播放** ⏸️

> 💡 **超爽设计**：当您按下 `1` 到 `5` 的任何一个键后，系统会**瞬间保存当前标注并自动为您播放下一首**！无需多点一次“确定”或“下一首”，实现真正的零等待切歌！

### 2. 鼠标点击
您也可以直接点击播放器下方的五彩大分类按钮进行打标。点击后同样会**瞬间自动切换到下一首**。

## 🤖 第三步：全自动 OCR 智能打标（无人值守，解放双手）

如果您已经把婴儿看护 App 的手机画面投屏到了电脑上，可以使用配套的 **Python 助手** 帮您完成全天候不间断的**全自动打标**：

1. **设置网页接收模式**：
   * 在网页左侧的“匹配模式”中，点击选中 **「2. 助手直连模式」**。此时网页会显示 *“正在等待连接...”*。
2. **连接手机并投屏**：
   * 用 USB 数据线将手机连接到电脑，确保开启了手机的 **USB 调试**。
   * 双击运行电脑上的投屏软件（如 `scrcpy`），将手机里的看护 App “历史报警界面”投屏在电脑屏幕上。
3. **启动 Python 助手**：
   * 在电脑的终端/命令行中，一键启动 Python 调度脚本：
     ```bash
     python baby_cry_ocr_helper.py
     ```
   * 启动成功后，网页端的连接状态会瞬时变更为 **`● 助手已连接`**（绿色高亮）。
4. **享受自动打标**：
   * 此时，每当网页放完一首音频，Python 助手就会自动通过 OCR 截屏读取手机投屏上的最新诊断结果，智能判定是否和当前播放时间吻合。
   * 确认吻合后，助手会**毫秒级自动直接将结果传回网页**，完成打标并自动切歌！您去喝杯茶的功夫，系统就已帮您标完了成百上千首音频。

## 💾 第四步：查看进度与导出成果

1. **实时保存**：
   * 您的每一次按键、每一次自动打标都会**实时安全写入硬盘**，即便中途电脑断电、死机或网页不小心关闭，已标注的历史成果也 **100% 绝对不会丢失**！
2. **导出结果**：
   * 随时可在页面最下方的“成果导出”板块中，点击一键下载：
     * **`下载标注 CSV`**：可直接使用 Microsoft Excel 或 WPS 软件打开，查看完美规整的文件路径、对应标签、播放次数以及确切打标时间戳。
     * **`下载标注 JSON`**：用于深度学习神经网络训练的结构化标准格式。
3. **一键重置**：
   * 如果想清空之前标错的数据、或是开启一轮全新的数据集打标，只需在右侧设置面板底部的“系统重置”一栏中，点击**“安全重置所有标注数据与文件池”**即可清空一新。

祝您标注愉快！如果在使用中遇到任何声音播放不流畅或连接异常问题，请优先检查您的浏览器硬件音频通道是否开启，或者重启 Python 脚本即可秒级恢复。
