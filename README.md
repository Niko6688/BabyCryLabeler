## 一、 开发者调试与本地运行指南

本系统支持 Windows、macOS 以及 Linux 跨平台运行。首次在本地部署或二次开发，请按以下步骤操作：

### 1. 终端/命令行工具的调用方式
* **Windows 系统**：按下键盘上的 `Win + R` 键，在运行窗口中输入 `cmd` 后点确定，即可调出黑色的 “命令提示符” 窗口。
* **macOS 系统**：按下 `Command (⌘) + 空格` 调出 Spotlight 搜索，输入 `终端` 或 `Terminal` 并回车。

### 2. 定位到项目文件夹 (使用 `cd` 指令)
由于程序需要进入项目对应的物理目录执行构建，需要确保终端的工作路径正确：
* **Windows 路径定位**（假设源码保存在 D 盘的 `BabyCryLabeler` 主文件夹中）：
  ```cmd
  D:
  cd D:\BabyCryLabeler
  ```
* **macOS 路径定位**（最爽捷方式：先在终端中输入下述命令前半段加一个空格，再从访达内**直接拖拽**该项目文件夹到终端窗口中，路径将自动生成）：
  ```bash
  cd /Users/您的用户名/Desktop/BabyCryLabeler
  ```

### 3. 一键安装依赖并运行
在进入对应的项目文件夹路径后，输入并执行以下脚本指令：
```bash
# 1. 自动同步拉取、安装项目所需的全部底层 npm 包模块与第三方编译库
npm install

# 2. 启动 Express 生产骨干网关与前端 Vite 的热重载本地服务
npm run dev
```
> [!IMPORTANT]
> 成功启动后，终端将输出对应的局域网物理访问链接，您可在浏览器（推荐使用最新版 Google Chrome 获得最流畅、原生的硬件渲染服务）中直接访问：👉 **`http://localhost:3000`**

## 三、 手机与电脑端物理连接指南 (Scrcpy 投屏)

整个全自动打标闭环的链路逻辑是：**电脑播放录音 ➔ 手机上的婴儿监护 App（如图灵看护 App 等）采听声音并发出警报 ➔ 电脑脚本实时截屏 ➔ OCR 字符匹配落盘**。

```
+-------------------------------------------------------------+
|                                                             |
|   1. 电脑端播放音频   -------->   2. 手机麦克风/对录线捕获   |
|          |                                     |            |
|          v                                     v            |
|   [ 自动标注系统 Web ] <-------   3. [婴儿监护App]判定分析  |
|          ^                                     |            |
|          | (API 同步/提交分类标签)              v            |
|   4. [Python 自动化 OCR] <====扫屏捕获==== [手机投屏画面]     |
|                                                             |
+-------------------------------------------------------------+
```

### 步骤 1：激活手机端开发者调试权限
1. 打开您的安卓手机（如华为、荣耀、小米、OPPO、一门等），进入 **设置 -> 关于手机**。
2. 连续点击 **“版本号”** 7次，直到系统弹出黑色提示 “您现在已处于开发者模式”。
3. 返回主设置菜单，找到 **系统和更新 -> 开发人员选项**，开启：
   * **USB 调试**
   * **USB 调试（安全设置 / 允许模拟点击）**（如有，请务必将其同步打开，用于后续可能的闭环互动与测试）。

### 步骤 2：拉起物理连接并运行 `scrcpy` 投屏
`scrcpy` 是全球最著名的超低延迟、免在手机端驻存流氓 App 的顶级开源手机画质流回传投屏神器：
* **Windows 端极速部署**：在命令提示符直接运行微软包管理器 `scoop install scrcpy` 或前往其官网 GitHub 下载离线包解压。
* **macOS 端极速部署**：直接打开 Terminal 键入大名鼎鼎的包管理工具命令：`brew install scrcpy`。
* **物理桥连**：用一根靠谱的高速 USB 线将手机物理接入电脑。手机亮屏并弹窗 “是否允许在此设备上允许 USB 调试”，勾选 **始终允许** 并点击确定。
* **双击启动**：在命令行中输入并在终端运行：`scrcpy`。几毫秒内，电脑桌面会弹出完美的安卓界面活动窗口，投屏顺利连通！

### 步骤 3：高保真音频环境路径推荐
* **扬声器方案（最省力操作）**：将电脑扬声器与麦克风置于同一个安静空间中，调大扬声器物理音量，促使手机能完全将高保真声波吸入分析。
* **物理内录反馈方案（深度推荐！深夜无人值守无声高品质专用）**：用最便宜的 3.5mm Aux 对录音频跳线将电脑的耳机物理输出口直接硬插接到安卓手机的音频接收孔上。这样在播放哭声时，电脑将完全无杂音，且没有外界的咳嗽声、风扇噪、或者家属说话声干扰，保证标注准确率达 99.9%！

## 四、 多模态标注内容智能匹配与识别逻辑

当手机上的婴儿监护 App 在听到哭声判定后，屏幕上对应的报警历史记录会生成如 `宝宝哭了 (饥饿)`、`宝宝哭了 (需要拍嗝)` 等指示牌。系统内置了三种截然不同的多模态匹配策略：

| 标注模式 | 操作交互类型 | 键盘热键 / 适配接口 | 适用实操场景 |
| :--- | :--- | :--- | :--- |
| **手动精细打标 (Manual Pattern)** | 人工 100% 盲听审听打标 | 主物理健 `1`、`2`、`3`、`4`、`5`（详见键盘区说明） | 适合复杂长尾样本深度鉴别与细微切片核对 |
| **高频剪贴板监听 (Clipboard Pattern)** | 半自动轮询拦截（1.5s 触发） | 在投屏端双击文字复制或借助 OCR 工具将诊断结果推入粘贴板 | 适合对定制录音、特殊投屏或模拟器进行快速手动划词打标 |
| **物理文件监测器 (File Monitor Pattern)** | 全自动无人值守数据吞吐 | 读取并在物理后台解析 `./demo_audios/result.txt` | 适合配合 Python 二次校验脚本实现高周转、大规模全流程自动标注 |

### ⌨️ 高频无物理鼠标干预键盘打标流（手动模式专属）
在进行人工精细化标注音频队列时，无需低头寻找鼠标，系统深度映射了标准键盘流：
* **主按分类键**：按下 **`1`**（饥饿）、**`2`**（不舒服）、**`3`**（犯困）、**`4`**（需要拍嗝）、**`5`**（烦躁/情绪）。按下即可秒级入库并**瞬时加载下一首**。
* **全局播放控制键**：按下 **`空格 (Space)`** 键。可在任何时候对当前播放中的音轨进行优雅的播放与暂停操作，辅助判听。

## 五、 Python OCR 辅助标注工作流配置

项目配套的 Python 自动化脚本 `baby_cry_ocr_helper.py` 是全自动流水线的“眼睛”和“总督导”。

### 1. 时间轴防错标精准闭环算法原理
传统的直接匹配会导致一个严重后果：由于图灵 App 本身神经网络推断有时滞（可能推延 3~10 秒），在切歌时，盲目截图可能会把几分钟前甚至是上一个录音产生的历史记录（如残留的“饥饿”）误打标给当前才播放了 2 秒钟的新音频。

**Python 助手通过时间比对解决了这个问题**：
1. 前端（Express）每开始播放、结束播放当前哭声音频，都会向后台写出高精度时间戳窗口。
2. Python OCR 截屏解析时，不仅会读取诊断文本（例如：`饥饿`），还会利用文字框周围的剪切布局，提取出这条历史报警所带有的小时/分钟。
3. 判定该警报的确切发生时机是否吻合在 **本首音频开播后的合理容忍物理窗口** 之中。
4. 如果数据异常、超时未出现新警报、或读取到的是陈旧废数据，一律向系统发起 **`skip` (安全跳过标注)** 指令。杜绝噪声。

### 2. 精准极速配置环境 (一行接一行)
1. 在项目所在的目录下打开命令行。
2. 运行包安装命令：
   ```bash
   pip install requests pillow opencv-python numpy pyperclip easyocr
   ```
   > [!TIP]
   > 如果您身处国内，网络下载由于外部带宽因素受到阻碍，请追加清华官方源镜像快速通畅安装：
   ```bash
   pip install requests pillow opencv-python numpy pyperclip easyocr -i https://pypi.tuna.tsinghua.edu.cn/simple
   ```
3. **环境故障救治**：若终端冷冰冰提示 `'pip' is not recognized as an internal or external command`，则是未添加 PATH 软链接，请重新运行 Python 安装包，并确保勾选底部的 **`Add Python to PATH`** 复选框。

### 3. 一键运行与全自动巡航指南
1. 进入本系统的 Web 界面，点击打开 **设置面板** (Settings)：
   * 将 **标注模式 (Labeling Mode)** 改写为：👉 **“API/结果文件轮询 (Result File/API Polling)”**。
   * 选择您期望的安全自动播放保护上限倒计时（推荐默认的 300 秒大宽容上限）。不用担心浪费时间，因为本方案是**高响应机制**，一旦 Python 后台捕捉报告匹配成功，便会**立刻切歌，无需被动等待倒计时结束**！
2. 将手机投屏窗口保持在桌面最顶层无遮挡（确保小时/分针与状态能被眼尖的 OCR 捕捉）。
3. 运行 Python 总调度引擎（它会自动识别平台、自动选择原生无声静音截图、自动完成多端口：3000/3124 重复兼容网关探测！）：
   ```bash
   python baby_cry_ocr_helper.py
   ```
4. 随后，点击 Web 主页上醒目的 **“播放第一首”** 大按钮。
5. **现在，开始享受您的咖啡时间 ☕**。数百首、甚至数千首音频数据样本，将通过扬声器/内录和手机 App 识别并在桌面上自行全自动、高精度地源源不断完成标签归类和极速落盘。

## 六、 网页版 (Web) 大容量音频导入攻略

由于浏览器内部的“黑盒沙盒安全属性（App Sandbox Security）”，任何一个标准的网页组件，在非宿主物理环境（Electron）下，均**无法绕过用户操作直接扫描你硬盘里的任意文件夹路径**。

为了保障零部署门槛，本系统提供了以下三种高品质解决姿势：

```
+--------------------------------------------------------+
| 方案A (演示用)     ➔ [ 自动合成本地模拟 Demo 音频 ]     |
| 方案B (极力推荐)   ➔ [ 拖入或点击一次性框选数百条音频 ]  |
| 方案C (局域网级)   ➔ [ 点击"扫描指定目录"并输入物理盘符 ] |
+--------------------------------------------------------+
```

* **方案 A：本地合成模拟仿真包（测试体验）**
  * 点击系统上的 “自动合成本地模拟演示音频 (Demo)” 标签，后端直接从服务器物理层面的 `./demo_audios/` 文件夹下为您吐出 5 首具有声源特征的高仿真声波，完美打通测试。
* **方案 B：拖拽与高维批量选择（极力推荐，最常用）**
  * 在 Web 主界面，直接一秒拖拽一个包含有数百条、数千条哭声多格式（支持 `.mp3`、`.wav`、`.m4a`、`.ogg`、`.flac`、`.aac` 等）的外部 Finder 窗口或 Windows 资源管理器文件夹到页面中心的拖拽区域。
  * 前端将结合标准的 `URL.createObjectURL` 技术为您快速生成虚拟解混流，保障浏览器沙盒内直接高效、极速不丢包持久化运行。
* **方案 C：输入完整绝对路径（强强对接方案）**
  * **在与服务器同台计算机运行时**，点击主控配置面板的 **“扫描指定路径”** 按钮。
  * 输入物理硬盘下的完整本地绝对地址（如 Windows：`D:\datasets\infant_cries`，或 Mac：`/Users/username/audio-dataset`）。网页服务端 Express 将以底层 Node 超神文件级别读取，并在 10 毫秒内完成数据索引注入！

## 七、 一键深度安全卸载与残留清理指南

* 🗑️ **应用程序（App Files）**：清除存放的所有物理二进制可执行客户端（macOS 端的 `.app`、Windows 端的 `unpacked` 绿色程序等）。

## 八、 高频热点常见问题 F.A.Q. & 故障排查

<details>
<summary><b>Q1：系统会因为我中途关闭应用，导致之前成百上千首已经辛辛苦苦标注的文件全部丢失吗？</b></summary>
<br>

**答：绝对不会！系统设计了金融级双持久重写防丢引擎：**
1. **客户端毫秒极速暂存 (LocalStorage Guard)**：

  1. **最直接的解决方式**：打开 **访达 (Finder) -> 应用程序 (Applications)**，在列表中找到 **`BabyCryLabeler`**，双击它运行一次。只要它打开过一次，macOS 系统就会自动将其登记并实时显示到您的启动台（Launchpad）中了。
  2. **强制重构启动台图标（最彻底）**：
     * 打开 Mac 自带的 **终端 (Terminal)**。
     * 复制并粘贴以下这行系统级 Dock/Launchpad 刷新指令，然后按下 **回车键**：
       ```bash
       defaults write com.apple.dock ResetLaunchPad -bool true; killall Dock
       ```
     * 屏幕和程序栏会瞬间闪烁刷新（非常安全正常），接着您再次打开启动台，软件的蓝色小图标就百分之百完美出现了！
