## 一、 Windows 平台完整使用教程与打包指引

本系统全套功能均在 Windows 10 / 11 操作系统上进行了极致适配与长效测试。您可以通过以下几种方式使用：

### 1. 网页开发者模式（最轻量推荐）
如果您仅希望用浏览器（如 Chrome, Edge）操作或共享局域网标注，请按照以下步骤运行终端命令：

* **第一步：如何打开终端**
  * **Windows 系统**：按下键盘上的 `Win + R` 键，在弹出的运行窗口中输入 `cmd`，然后点击“确定”或按回车，即可打开“命令提示符”窗口（黑色的终端界面）。
  * **macOS 系统**：按下键盘上的 `Command + 空格` 启动 Spotlight 搜索，输入 `终端` 或 `Terminal` 并回车开启。
* **第二步：定位到项目文件夹（使用 `cd` 命令）**
  * 当您打开终端后，需要切换到本系统解压之后的文件夹目录。
  * 例如，如果您的项目文件夹在 D 盘的 `BabyCryLabeler` 目录，您需要先输入 `D:` 并按回车（切换盘符），然后输入下述指令：
    ```cmd
    cd D:\BabyCryLabeler
    ```
  * 如果在 Mac 系统上，直接拖拽项目文件夹到终端窗口中，终端会自动生成它的完整路径，输入 `cd `（注意带空格）然后粘贴或回车：
    ```bash
    cd /Users/您的用户名/Desktop/BabyCryLabeler
    ```
* **第三步：运行服务命令**
  在项目文件夹下输入以下指令：
  ```bash
  # 1. 安装系统全套依赖项 (仅需在第一次运行时安装一次)
  npm install

  # 2. 启动本地热重载调试服务器
  npm run dev
  ```
启动后，直接在浏览器中访问：👉 **`http://localhost:3000`**

### 2. Electron 原生桌面版（开发预览）
如果您想把前端 Web 界面渲染为一个独立的 Windows 桌面窗口程序：
```bash
npm run desktop:dev
```

### 3. 一键编译打包生成 Windows 可执行安装包 (`.exe`)
为了让其他没有配置 Node.js/Vite/npm 运行环境的标注人员可以拆箱即用，您可以按照以下步骤将本套件完整打包：

1. **环境自检（仅限打包机）：**
   * 确保电脑中已成功安装 Node.js (推荐 LTS v18/v20+)，可在命令行运行 `node -v` 验证。
   * 将项目源码解压到**全英文路径**下，避免路径中含有特殊中文字符导致 Electron 打包工具解析路径报错。
2. **初始化并安装打包依赖（💡 极简首步，不可忽略）：**
   如果您是新解压的源码，打包前**必须先拉取项目所需的代码编译包依赖**，否则在下一步构建时会报错 `sh: vite: command not found`。请在同级终端中输入并回车：
   ```bash
   npm install
   ```

3. **执行一键打包命令：**
   在终端键入并回车：
   ```bash
   npm run desktop:build
   ```
   *小贴贴（Windows专用路径）*：若是在 Windows 电脑上，运行 `npm run desktop:build:win` 打出来的 `.exe` 会更完美。

   **打包逻辑背后的实现：**
   * 命令会自动执行 `npm run build`，利用 Vite 编译打包出超轻量、静态化的高性能前端网页资产到 `dist/`，并将后端 `server.ts` 利用 `esbuild` 预编译打包为 `dist/server.cjs`。
   * 接着，`electron-builder` 会调取 Windows 原生 NSIS 工具链，将 Electron 壳子与构建资产压缩打包。
4. **获取生成的安装/绿色文件：**
   打包成功完成后，根目录下会自动出现 `dist-desktop/` 文件夹。其中会包含：
   * **`win-unpacked/`** —— 免安装绿色直接运行版。您可以直接复制此文件夹给他人，双击其目录内的 `BabyCryLabeler.exe` 直接瞬间启动！
   * **`BabyCryLabeler Setup 1.0.0.exe`** —— 独立的 Windows 引导安装程序包（体积约为 60-80MB），支持发给任意没装开发环境的用户引导安装运行。

### 📥 在 Windows 电脑上安装与初次启动
1. 双击运行 `BabyCryLabeler Setup 1.0.0.exe` 安装文件。
2. **安装向导选项：** 
   * 系统支持用户自定义安装路径。您可以点击 "浏览" 自由选择非系统盘（如 D 盘）。
   * 勾选 "在桌面上创建快捷方式" 和 "在开始菜单创建快捷方式"（已在 `package.json` 中预配置好打包后默认自动创建）。
3. 点击 **下一步 / 安装**，系统会非常轻快地完成文件解压。
4. **启动运行：** 双击桌面上的蓝色 👶 图标，应用程序即会以优雅的桌面 Native 窗口完美呈现，您就可以彻底脱离浏览器，在纯手感、零干扰的主控看板中离线或在线操作全自动化标注工作了！

### 🛡️ macOS 独占注意事项、权限授予与硬件避坑指南
因为 macOS 独特的隐私安全防护机制，要成功实现闭环自动化匹配标注，**请务必手动设定并授予以下三项系统级权限**：

1. **屏幕录制与辅助功能权限（CRITICAL）：**
   * 后台运行 `baby_cry_ocr_helper.py` 脚本时需要对 `scrcpy` 手机窗口截图进行光学字符识别。
   * **如何设置**：请前往 **系统设置 -> 隐私与安全性 -> 屏幕录制**，勾选您所开启的终端应用（如 `Terminal`, `iTerm2` 或 `VS Code`）。
   * 同样在 **隐私与安全性 -> 辅助功能** 中允许对应的终端，否则 Python 脚本截图时会截出全黑画面而无法匹配！
2. **macOS 音频内录隔离与虚拟网关（对录避坑）：**
   * macOS 默认缺省内录混音驱动，电脑扬声器播放哭声时，手机麦克风在外部嘈杂环境采听准确率会下降。
   * **高品质静音方案**：建议用户在 Mac 上下载并使用免费开源的虚拟音频线路驱动 `BlackHole` (2ch) 或 `Soundflower`。在 Mac 的“声音”设置中将输出设置为 BlackHole，并将 `scrcpy` 或内录传输指向该通道，从而完美实现静音、高保真闭环数据吞吐。
3. **终端文件访问安全：**
   * 扫描指定大资源文件夹时，如路径为 `/Users/username/Downloads/dataset`，Mac 会在终端首个 API 触达时弹出“是否允许该应用访问您的下载文件夹/文稿”的隐私拦截，请必须点击 **“允许”**。
4. **macOS 自动截图免耳鸣静音参数：**
   * 我们的 Python 辅助脚本在 macOS 运行时采用原生 `screencapture -x` 命令实现底层静默截图，**彻底关闭了苹果系统烦人的默认咔嚓拍照音效**，保障在深夜或实验室高频运作时不对您和家人产生噪音干扰。

## 二、 手机与电脑端连接指南（Scrcpy 投屏方案）

本系统的一大核心功能，是让电脑扬声器播放婴儿哭声录音，让**手机上的婴儿监视器 App（例如：图灵看护 App 等）**直接听到并进行分析判定。随后通过 OCR 捕捉分析判定结果，完成全自动打标签。

以下是实现手机投屏并保持连接的详细流程：

```
+-------------------------------------------------------------+
|                                                             |
|   1. 电脑端播放音频   -------->   2. 手机麦克风/音频线捕获   |
|          |                                     |            |
|          v                                     v            |
|   [ 自动标注系统 Web ] <-------   3. [图灵看护App]分析警报  |
|          ^                                     |            |
|          | (API 同步/提交分类标签)              v            |
|   4. [Python 自动化 OCR] <====扫屏捕获==== [手机投屏画面]     |
|                                                             |
+-------------------------------------------------------------+
```

### 步骤 1：开启手机开发者调试权限
1. 打开您的安卓手机（如荣耀、华为、小米等），进入 **设置 -> 关于手机**。
2. 连续点击 7 次 “版本号”，系统会提示已开启 “开发人员选项”。
3. 返回主设置菜单，找到 **系统和更新 -> 开发人员选项**，开启如下两项：
   * **USB 调试**
   * **USB 调试（安全设置 / 允许模拟点击）**（如有，请务必保持开启，以支持模拟控制）

### 步骤 2：使用 USB 连接电脑并启动 `scrcpy` 投屏
`scrcpy` 是一款开源、延迟极低且无需在手机端安装任何 App 的手机画面投屏工具：
1. **下载或安装 `scrcpy`：**
   * **Windows 端：** 建议通过命令行包管理器安装：`scoop install scrcpy`，或者直接从 Github ([https://github.com/Genymobile/scrcpy](https://github.com/Genymobile/scrcpy)) 官方发布页下载打包好的压缩包并解压使用。
   * **macOS 端：** 运行 `brew install scrcpy`。
2. **连接验证：** 手机通过 USB 物理线连接到电脑。在手机屏幕出现的弹窗中勾选“允许在此电脑上进行 USB 调试”。
3. **启动投屏：** 在电脑命令行运行 `scrcpy`。此时，电脑屏幕上会弹出一个完全同步您手机画面的独立窗口，代表手机端连接成功！

### 步骤 3：物理音频环境配置
* **扬声器方案（最简单）：** 将电脑的网卡或声卡输出音量调至适当响度。将手机平卧于电脑音箱或麦克风采样器附近，确保手机麦克风能清晰、无刺耳杂音地采听到电脑音轨。
* **内录反馈方案（推荐高静音、无干扰场景）：** 使用标准的 3.5mm 对录线将电脑耳机孔直接连入手机音源插座；或在电脑端通过虚拟机/虚拟通道路由音频，这能从根本上消除外部高分贝哭声环境噪音对家人的打扰。

## 三、 标注内容智能匹配与识别逻辑

当手机接收到了电脑播放的哭声，手机上的“图灵看护 App”神经网络会分析出具体的警报，并在屏幕历史记录列表投射出醒目的警报词（如“宝宝哭了 饥饿”、“宝宝哭了 犯困”）。

本系统设计了三种数据匹配与最终写入规则：

### 1. 手动模式 (Manual Mode) — 效率专家
* 通过 Web 操作台中的 5 大高亮按钮（红色/饥饿、黄色/不舒服、蓝色/犯困、绿色/需要拍嗝、紫色/烦躁）手动选择。
* 键盘物理按键映射：在键盘按下**主按键 `1`（饥饿）**、**`2`（不舒服）**、**`3`（犯困）**、**`4`（拍嗝）**、**`5`（烦躁）**会立即触发对应标注逻辑，并自动唤醒下一首。适合人工高精度听审标注。

### 2. 剪贴板监听模式 (Clipboard Mode) — 半自动首选
* 本系统的前端页面启动了以 **1.5 秒为周期**的剪贴板轮询捕获。
* 当用户连接了其它非主流投屏甚至模拟器时，您只要利用 OCR 工具将识别的文本拷贝到系统剪贴板（或者用手机截屏自动识别复制内容），系统剪贴板状态一旦被检测到了相关敏感词：
  * `"hungry", "hunger", "饥饿"` ➔ 自动采纳为 **饥饿 (hungry)** 并扣发。
  * `"uncomfortable", "pain", "不舒服"` ➔ 自动采纳为 **不舒服 (uncomfortable)**。
  * `“犯困”、“需要拍嗝”、“烦躁”` 同理。
* 识别成功的音频，其标注文件会实时同步落盘。

### 3. 服务端文件自动轮询监听器 (File Monitor Mode) — 全自动无人值守
* 系统会实时、不间断地读取后台服务器上的接口与指定物理文件 `result.txt`（路径支持灵活设置，默认是 `./demo_audios/result.txt`）。
* 一旦有外部机器人或第三方图像检测工具分析成功后，将当前的婴儿状态英文/中文关键字覆写入该文档内，本系统将秒级截获其数值、将其赋予当前活动的音轨、并通知 Express 服务器清盘。

## 四、 Python OCR 辅助标注脚本运行教程

项目根目录中配套了一个功能强大的 python 本地 OCR 处理程序：`baby_cry_ocr_helper.py`。
此脚本可独立在电脑后台运行，用以扫描 `scrcpy` 投屏出来的手机画面、并配合内置防错机制、实现 100% 正确率的闭环智能全自动标注。

### 1. 精准防错标算法原理 (Timeline-based Anti-Mislabeling)
婴儿 App 的分析警报可能存在 1~3 分钟的模型滞后。如果您单纯匹配屏幕上现有的高亮词，很有可能会误提取到前几首留存在屏幕边缘的旧红点警告（如半小时前的“饥饿”记录）。
为此，Python 帮手采取了最严苛的**安全时间线窗口算法**：
1. 每当前置音轨播放完毕后，脚本自动锁死当前系统开始/结束的分秒时间戳。
2. 调取 Mac/Windows 超低噪音截屏机制，通过排版行垂直 Y 坐标切片，将每个检测到的关键字（如“饥饿”）与其隔壁的精确报警发生时间（如 PM16:39）一一匹配。
3. 计算该时间戳是否落在“本段哭声音频”开始播放后的安全误差段中。如果在，则确信是本条音频产生的结论；如果始终为历史浮点、或 4 分钟到期仍未匹配到新报警，则向 Express 服务器发出 **‘skip’ 安全跳过** 指令，规避弄脏数据集。

### 2. 安装 Python 核心依赖 (一步一步操作指南)
1. **打开您的电脑命令提示符 (CMD) 或 PowerShell / Terminal**（参照上文 `cd` 步骤，跳转到当前项目根目录中）。
2. 输入下述安装依赖的命令，并按键盘回车键：
   ```bash
   pip install requests pillow opencv-python numpy pyperclip easyocr
   ```
   * *小贴士 (针对国内用户)*：如果下载库的速度较慢，可以在命令后指定清华官方镜像站加速安装：
     ```bash
     pip install requests pillow opencv-python numpy pyperclip easyocr -i https://pypi.tuna.tsinghua.edu.cn/simple
     ```
3. **环境排错**：如果系统提示 `'pip' 不是内部或外部命令，也不是可运行的程序`，代表没有把 Python 环境变量添加到 PATH 中，可以在重新安装 Python 时，务必勾选底下最醒目的复选框：**`Add Python to PATH (将 Python 添加到系统变量)`**。

### 3. 智能跨平台与多端口自动探测服务

### Q2：运行 `npm run desktop:build:mac` 报错 `Missing script: "desktop:build:mac"` 是为什么？
* **根本原因**：这说明您在个人电脑上运行的是**旧版**本地代码，您的包配置文件 `package.json` 还没有我们新增的、针对 Mac 的专属自动化桌面打包脚本指令。

### Q3：执行默认 `npm run desktop:build` 报错 `wine64: bad CPU type in executable` 是怎么回事？
* **问题本质**：
  * 默认的 `desktop:build` 设置是针对 Windows (`--win nsis`) 进行编译的。
  * **在 Mac 电脑（尤其是苹果自研 M1 / M2 / M3 Apple Silicon 芯片系列上）**，跨平台将代码模拟构建为 Windows 下的 `.exe` 二进制文件，底层必须借助 `Wine64` 跨平台框架来运行微软的 `rcedit-x64.exe` 签名程序。
  * 由于 macOS 10.15 之后彻底停用了对 32 位底层程序的支持，并且苹果 ARM 架构不兼容未安装 Rosetta 2 兼容层的葡萄酒(Wine)虚拟机包，导致报出 `bad CPU type in executable`。
* **完美解法：**
  * **在 Mac 电脑上，打包出苹果原生支持的 `.dmg` 安装包**：
    执行您专属定制的：
    ```bash
    npm run desktop:build:mac
    ```
    *这一命令将完全跳过 Wine 和 Windows 分区编译。在您的 MacBook 上纯净完美地完成 Apple 本地打包逻辑，打出最精致且完全无错、原生流畅的 `.dmg` 桌面版安装软件。*
  * **在 Windows 电脑上，打包出微软原生的 `.exe` 安装包**：
    双击 Windows 电脑的端，在里面直接执行：
    ```bash
    npm run desktop:build:win
    ```
    *在宿主机器上，Windows 将以百分百的高效率直接完美输出 `.exe` 包文件，不报任何跨平台模拟警告。*
